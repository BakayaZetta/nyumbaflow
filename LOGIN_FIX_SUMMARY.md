# Login 500 Error - Root Cause and Resolution

## Problem Summary
User reported getting a 500 Internal Server Error when attempting to log in after the hardened PHP file sync. The authentication form would appear loaded but submissions would fail with a server error.

## Root Causes Identified and Fixed

### 1. Missing `login_attempts` Table (PRIMARY ISSUE)
**Problem:** The `rate_limit.php` module attempts to create the `login_attempts` table when loaded, but if the table doesn't exist initially, the CREATE TABLE statement error is silently caught. Subsequent rate-limiting queries would fail with a 500 error when the table doesn't exist.

**Solution:** Modified `db_config.php` to automatically create the `login_attempts` table immediately after establishing the database connection. This ensures the table exists before any code tries to use the rate limiting functions.

```php
// Added to end of db_config.php:
if ($pdo !== null) {
    try {
        $pdo->exec("CREATE TABLE IF NOT EXISTS login_attempts (
            id INT AUTO_INCREMENT PRIMARY KEY,
            ip_address VARCHAR(45) NOT NULL,
            action VARCHAR(50) NOT NULL,
            attempt_time DATETIME DEFAULT CURRENT_TIMESTAMP,
            INDEX idx_ip_action_time (ip_address, action, attempt_time)
        )");
    } catch (PDOException $e) {
        error_log("Warning: Could not create login_attempts table: " . $e->getMessage());
    }
}
```

### 2. Missing CSRF Token Validation (SECURITY ISSUE)
**Problem:** The `auth_action.php` accepts POST requests with CSRF tokens generated by the form, but it never validates them. This is a serious security vulnerability that could allow CSRF attacks.

**Solution:** 
- Added CSRF token validation to `auth_action.php` at the beginning of POST request handling
- Modified `csrf_token.php` to guard function definitions with `function_exists()` checks to prevent redeclaration errors

```php
// auth_action.php now validates CSRF token:
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $action = $_POST['action'] ?? '';
    
    // Validate CSRF token for all POST actions
    $csrf_token = $_POST['csrf_token'] ?? '';
    if (!verify_csrf_token($csrf_token)) {
        http_response_code(403);
        echo json_encode(['error' => 'Invalid request. Please try again.']);
        exit;
    }
    // ... proceed with action
}
```

### 3. Function Redeclaration Error (MINOR ISSUE)
**Problem:** The `csrf_token.php` file defines functions without checking if they've already been declared. If the file is included multiple times (which can happen in different workflows), it causes fatal errors.

**Solution:** Wrapped all function definitions in `if (!function_exists(...))` guards:

```php
if (!function_exists('generate_csrf_token')) {
    function generate_csrf_token() { ... }
}
// ... repeat for other functions
```

## Files Modified

1. **db_config.php**
   - Added automatic `login_attempts` table creation after DB connection
   - Ensures table exists before rate limiting functions are called

2. **auth_action.php**
   - Added CSRF token requirement from `csrf_token.php`
   - Added CSRF token validation before processing any action
   - Returns 403 Forbidden if token is missing or invalid

3. **csrf_token.php**
   - Wrapped function definitions in `function_exists()` guards
   - Prevents fatal errors from multiple inclusions
   - Makes the module more robust

    attempt_time DATETIME DEFAULT CURRENT_TIMESTAMP,
